Testes Web = Aplicações WEB - Sites

Automação WEB:
- Paga: Mercury QTP
- Open Source:  Selenium

Permite que repita os testes web quantas vezes for necessários
- Testes de Regressão

-Multiplus Browsers: Chrome, Opera, Firefox, Edge, Safari, etc
-Multiplus Sistemas Operacionais (O.S): Windows, Linux e OSX (MAC)
-Multiplas versões de Browser: Chrome (111), beta(112), alfa/canary.
-Resolução: muitas...

Varias Formas de organizar um projeto de Testes em WEB:

- Simples (SE IDE / manualmente)
- Selenium e Cucumber (BDD)
- Page Objetcs


O Selenium usa um Driver para se comunicar com o Browser.
Código(selenium) -- Driver -- Browser

No caso do Chrome/Edge: 1 para cada versão.
Nos demais, normalmente, 1 versão de driver funciona para várias do navegador.

Há 3 formas de configurar/utilizar um driver:
- Dowload Manual e instala dentro do projeto. (Bom: Portátil - Ruim: Manutenção)
- Dowload manual e instala em uma pasta compartilhada. (Bom: Manutenção - Ruim: Portátil)
- Gerenciador de download automático. (Bom: Manutenção + Portátil - Ruim: Usa Recursos)

Cucumber
- Framework para associar documentos de negócio (.feature) como arquivos de automação.
- Plugins Gherkin e Cucumber
- Permite que cremos especificações e critérios de aceite em Gherkin
- Esse tipo de especificação é recomendado dentro da abordagem BDD (Desenvolvimento Direcionado por Comportamente)

Page Objects / Page Factory
- Automação de teste robusta, que simplique a manutenção e ampliação do teste
- Programação Orientado a Objetos
- Permite que criemos o mapeamento dos elementos separados dos scripts de automação
- Podemos usar com ou sem as especificações em Gherkin

Exemplos:
sem Gherkin
- Um arquivo de mapeamento para cada página ou seção repetida entre página (cabeçalho/ rodapé / menu)
- um arquivo de automação do fluxo de teste, que pode usar o mapeamento de páginas diferentes

Com Gherkin
- Um arquivo de mapeamento para cada página ou seção repetida entre página (cabeçalho/ rodapé / menu)
- um arquivo de especificação dizendo o fluxo desejado
- um arquivo de passos que segue a especificação e usa o mapeamento.

Formas de Automatizar:
0 - Gravação de Script (1 arquivo)
1 - Script simples (1 arquivo)
2 - Cucumber (2 arquivos: feature e steps)
3 - Page Object Simples (2 arquivos: mapeamento e scripts)
4 - Page Object + Cucumber (3 arquivos: mapeamento, feature e steps)

3 Desafios na Automação
- Identificação de Seletores (nem sempre ID/Name, então vamos de CSS / Xpath)
- Sincronismo (ajustar a velocidade entre script e software alvo)
- "Programação Exótica": inovadora ou muito ruim.
